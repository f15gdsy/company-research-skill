# company-research-skill

用「三个问题 × 五个动作」框架快速拆解一家陌生公司（方法来源：[【@迪迪碎碎念_AI】](https://www.bilibili.com/video/BV1mqLJ6CERW/)）。核心不是堆数据，而是**用对的问题把用户推到独立判断的阶段**。

## 框架总览

| 阶段 | 核心问题 | 目的 | 动作 |
|------|---------|------|------|
| 第 1 小时 | 这家公司是谁？ | 建立坐标，搞清定位、行业特点和收入结构 | 【1】 类比 → 【2】 画业务版图 → 【2.5】 扫描行业特点 |
| 第 2 小时 | 凭什么活着？ | 找到真正的护城河 | 【3】 找命脉 |
| 第 3 小时 | 会不会死？ | 看风险，防被好消息洗脑 | 【4】 反方辩手 → 【5】 形成判断 |

三个问题的顺序不能换。先搞清楚公司是谁，再谈护城河，最后看风险。

## 核心设计原则

- **AI 不是替用户思考**，而是把资料快速摊开，让用户更快进入判断层
- **不是研报工具**：产出是"判断"，不是 50 页报告
- **允许不完美**：信息永远不完备，标注"暂缺"继续
- **判断优于数据**：宁可有偏的判断，不要完美的数据搬运
- **证据等级分离**：五级证据体系（A 级-公司披露 → E 级-AI 推理），语气强度绑定证据等级

## 数据纪律

- **五级证据体系**（§1.3）：A 级（公司披露）到 E 级（AI 推理），语气强度与证据等级硬性绑定
- **反幻觉约束**（§1.4）：良率、客户份额、ASP 等易幻觉字段，无公开支撑必须标注"未公开披露"
- **交叉验证**：营收、净利润、增速、用户规模四大指标必须从至少两个独立来源验证；同指标差异 >10% 须显性化暴露
- **数据源降级**：neodata-financial-search（首选）→ Tavily 搜索 → WebSearch → WebFetch

## 输出格式

分析完成后创建 Obsidian 笔记（路径：`30-Knowledge/公司研究/[公司名] - 公司拆解 [YYYYMMDD].md`），含结构化模板：一句话定性、业务版图、护城河分析、风险清单、独立判断、可信度摘要、输出前检查清单。
